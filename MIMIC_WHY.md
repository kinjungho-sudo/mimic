## 1. WHY — 우리는 왜 만드는가

### 출발점이 된 관찰

B2B SaaS 기업의 CS팀에는 매일 반복되는 장면이 있다.

> 신규 고객이 입사한다. CS 담당자가 1시간짜리 온보딩 콜을 잡는다.  
> 2주 후, 또 다른 신규 고객이 입사한다. 같은 콜을 다시 잡는다.  
> 한 달에 10번. 1년에 120번. 매번 같은 내용을.

이건 CS 담당자가 게으르기 때문이 아니다. **배우는 사람을 위한 제대로 된 도구가 없기 때문이다.**

기존에 있는 것들:
- PDF 매뉴얼 → 아무도 안 읽는다
- PPT 가이드 → 만드는 데 너무 오래 걸린다
- 영상 튜토리얼 → 제작은 지옥, 수정은 더 지옥

결국 "사람이 직접 설명하는 것"이 가장 빠르고 확실한 방법으로 남는다.  
MIMIC은 이 "사람이 직접 설명하는 것"을 한 번만 하면 되도록 만든다.

---

## 2. WHO — 누구를 위해

### 1차 타겟: CS팀 (고객 지원, 온보딩 담당자)

- **기업 규모**: 시리즈 A~B, 20~100인 B2B SaaS 기업
- **직무**: 고객 온보딩 매니저, CS 매니저, 기술 지원 담당자
- **일상**: 같은 기능을 다른 고객에게 반복 설명, 신규 고객사 담당자 교체마다 재교육

### 왜 CS팀인가

CS팀은 두 가지 고통을 동시에 갖고 있다.

1. **만드는 고통**: 교육 자료를 만들 전담 인력이 없다. 결국 CS 담당자가 직접 PPT를 만들거나 영상을 찍는다.
2. **반복하는 고통**: 만들어도 어차피 "한번 직접 보여주는 게 빠르다"가 된다.

MIMIC은 이 두 고통을 동시에 해결한다.

## 3. WHAT — 어떤 문제를 해결하는가

### 데이터로 본 현실

| 데이터 | 출처 |
|--------|------|
| CSM의 **66%**가 하루의 상당 시간을 반복 행정 업무에 소비 | Vitally 2025 CS Confidence Index |
| CSM의 **72%**가 자동화하고 싶은 업무가 있다고 응답 | Vitally 2025 |
| 직장인 전체 근무 시간 중 **62%**가 반복 업무 | Clockify 2025 Research |

### CS팀 반복업무의 두 종류

이 데이터에서 중요한 건 **"반복업무"가 두 종류로 나뉜다**는 점이다.

```
B타입 — 자동화 대상 (사람이 안 해도 되는 것)
  · 온보딩 이메일 발송
  · 갱신 알림 / 체크인 리마인더
  · 헬스스코어 모니터링
  · 시스템 간 데이터 싱크
  · QBR 일정 자동 알림
  → 이미 Gainsight, ChurnZero, n8n, Teach Claude가 해결 중

A타입 — 반복 교육형 (사람이 매번 설명해야 하는 것)
  · 신규 고객 제품 온보딩 시연
  · 동일 기능 반복 설명
  · 신입 CS팀원 교육
  · 같은 문의 반복 답변
  · 신규 기능 내부 전파
  → 제대로 된 솔루션이 없다
```

**MIMIC은 A타입 문제를 해결한다.**

B타입은 이미 레드오션이다. A타입은 아직 제대로 된 솔루션이 없는 영역이다.

## 4. HOW — 어떻게 해결하는가 (5레이어 구조)

### 전체 아키텍처

```
L5. AI 에이전트 자동화  ← 비전 / 장기 해자
    캡처된 좌표 = AI 실행 좌표 / 사람 시연 → AI 무한 반복
      ↑
L4. 인터랙티브 뷰어  ← 현재 핵심 차별점
    Guideflow형 / 클릭으로 따라하기 / 행동으로 기억
      ↑
L3. AI 편집기  ← 제작 시간 단축
    어노테이션 자동 생성 / 설명 자동 작성
      ↑
L2. 워크스페이스  ← 팀 협업 인프라
    개인/팀 / 폴더 / 권한 / 공유
      ↑
L1. 캡처 엔진  ← 진입점
    Chrome Extension / DOM+좌표 / PII 블러
```

### 레이어별 벤치마킹과 우리의 차별점

| 레이어 | 벤치마크 | 우리가 가져온 것 | 우리만의 것 |
|--------|---------|----------------|-----------|
| L1 캡처 | Scribe, Tango AI | 사이드 패널, PII 블러 | DOM 셀렉터 + Vision 하이브리드 (L5 준비) |
| L2 워크스페이스 | Scribe, StepHow | 팀 워크스페이스, 폴더, 권한 | (동일 수준, 차별 없음) |
| L3 편집기 | Tango AI, StepHow | AI 자동 어노테이션, 설명 생성 | Claude API 기반 한국어 고품질 |
| L4 뷰어 | **Guideflow** | 인터랙티브 슬라이드쇼, 클릭 유도 | 스텝별 AI 음성/자막, 한국어 최적화 |
| L5 자동화 | Claude in Chrome, Tango AI | 자동화 개념 | **아무도 완성 못 한 영역 — 우리가 먼저** |

## 5. VISION — 어떤 제품이 되려 하는가

### 단기 비전 (1년)

> **"CS팀의 A타입 반복 업무를 없애는 도구"**

CS 담당자가 신규 고객에게 제품을 설명하는 방식을 바꾼다.  
직접 콜을 잡는 대신 → MIMIC 링크 하나를 보낸다.  
그 링크 안에서 고객은 클릭으로 직접 따라하면서 배운다.

### 중기 비전 (2~3년)

> **"업무 패턴 학습 플랫폼"**

사람이 시연하는 과정에서 캡처된 데이터는 단순한 스크린샷이 아니다.  
DOM 셀렉터 + 클릭 좌표 + 실행 순서 = **업무 패턴의 디지털 기록**이다.

이 데이터가 쌓이면:
- AI가 동일 작업을 자동으로 반복 실행
- 조직의 업무 노하우가 시스템에 축적
- 신입이 들어오면 AI가 가이드 역할

### 장기 비전 (L5)

> **"AI 에이전트가 인간의 클릭/워크플로우를 복제하는 자동화 인프라"**

MIMIC이 수집한 데이터(DOM 셀렉터 + 좌표 + 순서)는  
AI 에이전트가 동일 작업을 실행하는 데 필요한 정확한 좌표다.

사람이 한 번 시연하면 → MIMIC이 가이드를 만들고 → AI가 무한히 반복한다.

이것이 경쟁사가 아직 완성하지 못한 영역이고,  
우리가 L1부터 이 구조를 염두에 두고 데이터를 설계하는 이유다.
