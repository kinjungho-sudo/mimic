이번 작업은 단순 버그 수정이 아니라 Parro Desktop Capture를 Scribe에 가까운 클릭 기반 workflow recorder로 재설계/개선하는 작업입니다. 아래 브리프와 worklog 규칙을 구현 기준으로 사용하세요.

- docs/codex/CODEX_BRIEF_v0.7_scribe-level-desktop-capture.md
- docs/codex/CODEX_WORKLOG_PROTOCOL.md

핵심 목표:

사용자가 Desktop Recording을 시작한 뒤 데스크톱 앱/웹/OS 화면에서 클릭하면, 클릭한 화면만 자동 캡처되고, 클릭 위치가 하이라이트로 표시되며, 캡처된 step이 Parro 사이드 패널에 즉시 쌓이고, 완료 버튼을 누르면 자동으로 매뉴얼 생성/편집 화면으로 이동해야 합니다.

반드시 지켜야 할 기준:

1. Desktop Capture는 전체화면 스크린샷 기능이 아니라 Scribe-like click-driven workflow recording이어야 합니다.
2. 녹화 중 사용자가 클릭할 때마다 step이 자동 생성되어야 합니다.
3. 듀얼 모니터 환경에서 양쪽 화면 전체를 한 장으로 캡처하면 안 됩니다.
4. 클릭한 화면만 캡처한다 — 클릭이 발생한 display 또는 가능하면 window/app 영역만 캡처해야 합니다.
5. 클릭 좌표는 virtual desktop 기준이 아니라 캡처된 이미지 기준 clickXPct/clickYPct 또는 동등 필드로 변환되어야 합니다.
6. 캡처된 이미지에는 클릭 위치 하이라이트가 보여야 합니다. 기존 annotation 시스템이 있으면 marker/spotlight/recorderBox를 재사용하세요.
7. 캡처 중 Parro 내부 side panel/step panel에 thumbnail preview가 즉시 추가되어야 합니다.
8. 완료 버튼을 누르면 finalize API를 호출하고, 생성된 manual/tutorial id를 받아 manual editor 또는 현재 제품의 매뉴얼 제작 화면으로 자동 이동해야 합니다.
9. 사용자가 완료 후 별도로 매뉴얼 생성 버튼을 찾아 누르게 만들면 안 됩니다.
10. OS 권한이 없거나 desktop runtime이 없으면 그 사실을 숨기지 말고, 가능한 fallback과 정확한 blocker를 보고하세요.
11. 작업 후에는 반드시 `docs/worklogs/codex/YYYY-MM-DD_desktop-capture-scribe-parity.md` 형식으로 worklog를 남기세요.
12. worklog에는 읽은 파일, 변경 파일, 구현 내용, 검증 명령과 실제 출력, 수동 QA 결과, 미검증 항목, blocker, MAX가 검토해야 할 다음 액션을 포함하세요.
13. 가능하면 GitHub branch에 commit/push하고 commit hash 또는 PR URL을 보고하세요.
14. 채팅 보고만 하고 repo worklog를 남기지 않으면 작업 완료로 보지 않습니다.

먼저 해야 할 조사:
