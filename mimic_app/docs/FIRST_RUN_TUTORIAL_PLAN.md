# 첫 사용자 시작 튜토리얼 계획

## 문제

첫 사용자는 빈 워크스페이스에 진입한 뒤 `화면 녹화`, `직접 편집`, `새 플레이북` 중 무엇부터 해야 하는지 알기 어렵다.

## 목표

첫 사용자가 별도 설명 화면을 읽는 데서 끝나지 않고, 실제 제품 화면 위 Live Guide를 따라 첫 녹화 흐름을 시작하도록 안내한다.

## MVP 범위

1. 기능 도입 이후 가입했고 개인 워크스페이스에 매뉴얼이 없으며 튜토리얼 이력이 없는 사용자에게 Live Guide 환영 연출을 자동 표시한다.
2. 환영 연출이 끝나면 실제 `새로 만들기`와 `새 매뉴얼(녹화)` 버튼을 차례로 스포트라이트 처리한다.
3. 사용자가 강조된 실제 버튼을 클릭해야 다음 단계로 진행한다.
4. 기존 `RecordingModal`에서도 `페이지 선택하기` 버튼을 강조해 Recorder 흐름으로 연결한다.
5. 튜토리얼은 언제든 건너뛸 수 있고, 빈 상태에서 다시 시작할 수 있다.
6. 튜토리얼을 시작한 사용자가 녹화를 마치고 편집 화면에 들어오면 완료 안내를 표시한다.
7. 상태는 사용자 ID가 포함된 브라우저 저장소 키에 보관한다.

## 제외 범위

- 랜딩페이지 데모 및 영상
- 플레이북 튜토리얼
- Recorder 녹화 로직 변경
- Supabase 스키마 변경

## 완료 기준

- 기존 사용자와 팀 워크스페이스에는 자동 노출되지 않는다.
- 각 단계에서 실제로 눌러야 할 행동 하나만 밝게 강조된다.
- 건너뛰기 후 강제로 다시 열리지 않는다.
- 녹화 완료로 열린 편집기에서 첫 매뉴얼 완성을 확인할 수 있다.
- 키보드 포커스와 모바일 레이아웃을 지원한다.
