절대 지켜야 할 작업 원칙
이 6가지를 어기면 작업 손실 발생 — 작업 시작 전 반드시 숙지
⚠ Critical Working Rules

"이미 만들어진 것은 절대 재작업하지 않는다"

  • 원칙 1. 이미 작동하는 페이지·컴포넌트는 그대로 유지한다. 디자인이 마음에 안 들어도 김정호 승인 없이 갈아엎지 않는다.
  • 원칙 2. 모든 새 파일은 TypeScript (.tsx)로 작성한다. 기존 .js 파일을 만났을 때만 .tsx로 변환한다.
  • 원칙 3. 디자인 토큰(CSS 변수 --mm-*)을 반드시 사용한다. 색상 hex 값을 직접 쓰지 않는다.
  • 원칙 4. 백엔드 API가 아직 없을 때는 가짜 데이터(mock)로 UI를 먼저 완성한다. 나중에 API만 연결한다.
  • 원칙 5. shadcn/ui 컴포넌트가 있으면 그것을 우선 사용한다. 새로 만들지 않는다.
  • 원칙 6. 모든 페이지는 반응형으로 작성한다. 특히 플레이어는 모바일 시청 가능해야 한다.

마스터 클럭 = 음성 audio의 timeupdate
useEffect(() => {
  const audio = audioRef.current;
  const handler = () => {
    const ms = audio.currentTime * 1000;
    markers.forEach(m => {
      if (ms >= m.script_offset_ms && !triggered.has(m.id)) {
        triggerMarker(m); // 커서 이동 + 자막 + 사운드
        triggered.add(m.id);
      }
    });
  };
  audio.addEventListener('timeupdate', handler);
  return () => audio.removeEventListener('timeupdate', handler);
}, [markers]);
오차 목표: 100ms 이내. 음성 OFF일 때는 단계당 5초 자동 진행.

POST /api/extension/link 1회용 토큰 발급 (5분 유효)
▸ 매뉴얼 데이터 API
GET /api/tutorials 내 매뉴얼 목록
GET /api/tutorials/[id] 매뉴얼 상세 (steps + markers 포함)
PATCH /api/tutorials/[id] 매뉴얼 수정 (자동 저장)
DELETE /api/tutorials/[id] 매뉴얼 삭제
POST /api/tutorials/[id]/publish 게시 → share_token 반환
▸ AI 자동 생성 API
POST /api/generate-script 사용자 초안 → Claude → 다듬어진 스크립트
POST /api/generate-markers step들 → Claude → 마커 자동 배치
POST /api/generate-annotations AI 프롬프트 → Claude → 효과/주석 적용
POST /api/tts 텍스트 → OpenAI TTS → mp3 URL
