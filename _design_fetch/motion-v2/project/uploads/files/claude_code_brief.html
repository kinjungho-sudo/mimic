절대 지켜야 할 작업 원칙
이 7가지를 어기면 보안 사고 발생 — 작업 시작 전 반드시 숙지
⚠ Critical Working Rules

"클로드 디자인이 막히지 않게 백엔드를 먼저 만든다"

  • 원칙 1. 이미 작동하는 middleware.ts, lib/supabase.ts, lib/supabase-server.ts는 절대 갈아엎지 않는다. 필요시 확장만.
  • 원칙 2. Service Role Key는 서버 사이드(API Route)에서만 사용. 클라이언트 코드 import 금지.
  • 원칙 3. 모든 API 입력은 zod 스키마로 검증한다. 입력 검증 없는 API 라우트 금지.
  • 원칙 4. 모든 테이블에 RLS 정책 필수. RLS 없는 테이블 만들면 데이터 유출.
  • 원칙 5. 외부 API (Claude·OpenAI) 호출은 반드시 백엔드 경유. 클라이언트에서 직접 호출 금지.
  • 원칙 6. 모든 API에 인증 가드 + 에러 핸들링 + Rate Limiting 적용.
  • 원칙 7. 클로드 디자인이 호출할 함수는 lib/api/* 또는 lib/auth-client.ts로 미리 만들어 둘 것.

NA_steps (기존)
Naviaction 캡처 원본 — 보안 수정 필수
CRITICAL
🚨 즉시 처리
RLS 비활성화 + user_id 누락
현재 누구나 NA_steps SELECT/INSERT/UPDATE/DELETE 가능한 상태. 아래 순서대로 마이그레이션 작성.
▸ 마이그레이션 단계
-- Step 1: user_id 컬럼 추가 (nullable)
ALTER TABLE NA_steps ADD COLUMN user_id uuid NULL;

-- Step 2: 기존 row를 정호씨 user_id로 일괄 업데이트
-- (정호씨 user_id는 환경변수 또는 별도 입력)
UPDATE NA_steps SET user_id = '정호씨_user_id'
WHERE user_id IS NULL;

-- Step 3: NOT NULL 적용 + FK 추가
ALTER TABLE NA_steps
  ALTER COLUMN user_id SET NOT NULL,
  ADD CONSTRAINT NA_steps_user_fk
  FOREIGN KEY (user_id) REFERENCES auth.users(id);

-- Step 4: RLS 활성화
ALTER TABLE NA_steps ENABLE ROW LEVEL SECURITY;

-- Step 5: 본인 데이터만 CRUD
CREATE POLICY "na_steps_own" ON NA_steps
  FOR ALL USING (auth.uid() = user_id);

-- Step 6: tutorial_id 컬럼 추가 (MM_tutorials와 연결)
ALTER TABLE NA_steps ADD COLUMN tutorial_id uuid NULL
  REFERENCES MM_tutorials(id) ON DELETE SET NULL;

AI 자동 마커 생성 프롬프트 (Claude)
다음 매뉴얼 데이터를 보고 ①②③ 마커를 자동으로 배치해줘.

스텝 목록:
{steps에 대한 title, description, page_url, click_x, click_y, screenshot_url}

규칙:
- 모든 클릭 위치 = 마커 후보
- 너무 가까운 클릭은 하나로 묶기
- 사용자 시선 흐름 자연스럽게 (좌→우, 위→아래)
- 마커 번호는 시간 순서대로

응답 형식 (JSON만):
{
  "markers": [
    {
      "step_id": "uuid",
      "marker_number": 1,
      "position_x": 0.18,
      "position_y": 0.38,
      "connected_effects": ["click_sound"]
    }
  ]
}
마커 시간 자동 매핑 함수 (lib/markers.ts)
export function calculateMarkerTimings(
  script: string,
  audioDurationMs: number,
  markers: Marker[]
): Marker[] {
  // 스크립트에서 ①②③ 위치 찾기

백엔드가 디자인 작업을 막지 않도록 보장하는 규칙
▸ 핸드오프 순서
  • 1순위: 타입 정의 (types/index.ts) — 디자인이 가짜 데이터 만들 때 참조
  • 2순위: lib/auth-client.ts — 로그인/가입 페이지 작업 시작 가능
  • 3순위: lib/api/tutorials.ts — 대시보드/에디터 데이터 fetch 가능
  • 4순위: lib/api/ai.ts — 에디터 AI 재생성 작업 가능
  • 5순위: lib/api/events.ts — 플레이어 이벤트 로깅 가능
